Previous Releases

This Previous Release page provides the user the ability to revisit what functionality was updated and when it was updated. This could be helpful if a user had used mGIS in the past, had not used it in a while, but was using it again.

.

.

May 3, 2021 Release

The May 3, 2021 release sees the change of mGIS to mPATH. In addition to the functionality listed below, underlying updates were made to improve performance and navigation.

Filters now function per program

Previously, selecting filters in a program (for example, “Program A”), set the filters for that program, as expected. However, if the user selected another program, filter settings from “Program A” were still set and affected the filter results in the other program. Now, filters are set per program and reset automatically as the user changes programs.

Stage drop-down changed to graphical selection

Previously, Task, Issue, and Risk stages were selected through a dropdown menu. Since these selections represent Kanban stages, the dropdown menu was replaced with a graphical step component giving the user a visual representation of their progress through Kanban stages.

Last Update column sorts by Date and Time

Previously, the Last Update column grouped all Last Updates together in no particular order. Now, users can sort this column by date and time of Last Update.

Risk Approval section user interface and workflow simplified

Based on requests to modify the Risk Approach Approval workflow, we have drastically simplified the Risk Approach Approval process by giving any permissioned user the ability to approve the Risk Approach. Additionally, several fields that were previously required to save the risk form are no longer required, making form completion less burdensome.

Favorites tab added in Advanced Filters

Users can now name and save their filter settings in a "Favorites" tab. The Favorites tab features a dropdown menu that lists the user’s Favorites based on saved filter settings. Additionally, admins can share their Favorites. This gives users the ability to apply pre-selected filter settings (i.e., Favorites) saved by others.

Breadcrumbs are clickable

The title section of the forms were recently modified. The action bar was replaced by a breadcrumb-style top section containing the project name, current action (ie, Task, Issue, or Risk), and the action title. In this release, the breadcrumb-style title section now features active links to further aid the user in navigating back to the action tab or project with one simple click.

Major upgrade to mPATH navigation for Sheet, Map, and Kanban views

Previously, refreshing the page anywhere in client panel would restart the user’s session in their last selected tab. This release includes a major upgrade to mPATH navigation routes which provides user with noticeable navigation flow efficiency across views and tabs.

February 23, 2021 Release

The February 23, 2021 release was primarily focused on improving the user's ability to use mGIS. Functionality was enhanced to be able to move and duplicate tasks. Reporting on Checklists was improved. Risk management was further enhanced. User's can define mGIS start up preferences. Some performance enhancements were included as well.

Improvements
Added the ability to provide the Progress of a Checklist item

Checklist items were updated to take up less space so that more checklist items can be seen. Additionally, checklist items can be expanded to see the details of the item.

Most importantly, checklist items have been updated to provide the user the ability to add the progress that they are making on the items they are working on within a Checklist. To add a progress update, the user will click on the + next to Progress Update.

After selecting the + next to Progress Update the user will see where they can enter their progress update.

After the user saves, the Progress Update will be added to the Progress table.

Enhanced duplicate and move functionality

The ability to Move and Duplicate items has been consolidated to a right-click menu off of the Sheet table.

  • Open will open the details page of the item same as double-clicking on the row.

  • Duplicate will create a duplicate within this same project with - Copy appended to the name of the item.

  • Duplicate To ... will open another drop-down menu where the user can select to where they want to duplicate the item.

  • Move To ... will open another drop-down menu where the user can select to where they want to move the item.

  • Delete will delete the item after confirming that is what the user wants to occur.

Provided ability to have a Risk Approach approved

The Risk Prioritize page was updated to provide the ability to have a Risk Approach approved.

The assigned Risk Approach Approver must be signed in to approve the approach. Upon selecting the Approved checkbox the Date and Time of the approval will be captured.

Created a user preferences page

Different users can have different responsibilities associated with their use of mGIS. So to assist users in focusing mGIS on where they want to start their work, Preferences and Start On have been added to their User Profile.

The user can bypass the Welcome page by selecting the Program they wish always start their work. If the Program is selected, the user can select one of the Project Groups within the Program, and if a Project Group is selected the user can select the Project they wish be opened first.

Additionally, the user can default their initial Navigation to opening on one of the views: Map, Sheet, Kanban, Gantt, or Team.

Lastly, the user can default their Sub Navigation to be one of the tabs: Overview, Tasks, Issues, Risks, or Notes.

Updated Assigned Users on the Sheet table to have the appropriate R or A next to the user's name

So that the users can tell the role of the assigned users from the Sheet table, R and A were added in front of the user who is Responsible and Accountable.

The roles of Consulted (C) and Informed (I) were not included to save on the landscape of the page and not user to many rows. However, in a near future release a check box will be added to this page that will allow the user to show the C and I users as well when needed.

Updated the Last Update column to distinguish the meta data from the Update

So that users can more easily distinguish the last update from the time and date of the last update, the time and date have been distinguished.

To save on landscape in the table, who made the last update can be seen by hovering over the date and time.

Replaced Modal details edit page associated with the Sheet view

Due to the modal details edit page causing so many issues, it was removed and replaced with an inline edit page.

The Save button no longer closes the page, so the user can now save their changes along the way without have to reopen the details page. To close the details page select the Close button.

Issues Resolved
Project not defaulted to the current project

The Project field was added as an option to be able to move an item to somewhere else. The primary problem with this field was that when adding a new item it was defaulted to the current project. Anyway, better functionality to move items has been put into place so this field has been removed. See Enhanced duplicate and move functionality under Improvements above to additional details on the new move functionality.

Rounding issue

Calculations did not have a rounding rule, so only used the numbers before the decimal place. Standard rounding rules, .5 round up, otherwise round down, are now used.

The delete of a Risk, does not delete the Risk

There was a bug so the delete did not complete, but this has been resolved.

Not able to set Risk "On Watch"

There was a bug such that setting On Watch was not captured and that has been resolved. There is another bug that selecting On Watch closes the page that still needs to be resolved.

On Risks, Probability Description needs a * to show it is required

The * has been added to show the field is required.

Add Task on Map did not display the add form

There was a bug that has been resolved.

Search Risks on Sheet page never found results

There was a bug that has been resolved.

Added Related Risks to Issues

The ability to add related risks to issues was missing and was added.

February 5, 2021 Release

The February 5, 2021 release was primarily focused on improving the user interfaces that are associated with performing CRUD (Create, Read, Update, Delete) within mGIS. The Overview / Summary / Stats pages were updated for easier readability and the addition of requested information. The Map of the Map view was updated to resolve some inconsistencies, but more work is needed to maintain the context of the map as mGIS is being used. Some basic duplicate and move functionality was introduced, but more advanced functionality for copy / move will be include in future releases. Some performance enhancements were done as well.

Online Help Home page with access to the release plans

A new online help functionality is being added to mGIS. Though still under construction, this functionality can be access by selecting Help on the main menu in the upper right corner of mGIS. Selecting Help will invoke the mGIS Online Help site:

From this site the user will be able to access:

  • The Admin Panel User's Guide (still under construction)

  • The Client Panel User's Guide (still under construction)

  • What's New in the current release

  • What's Coming in future releases

  • What was included in Previous Releases

Rename Projects to Programs and Facilities to Projects

To provide a more broad usage of mGIS:

  • all references to Project(s) were changed to Program(s)

  • all references to Facility / Facilities were changed to Project(s)

Update the workflow to be consistent across CRUD (create, read, update, delete) activities

The Facility Manager has been deprecated because the Sheet view has been the preferred view of users.

The details pages for Tasks, Issues, and Risks under the Map and Sheet views have been updated for consistency. Additionally, these pages have been updated to improve the user experience by grouping the information under tabs so less if any scrolling is necessary.

Update implementation of records shown per page to be consistent throughout

All the tables throughout mGIS now have a consistent look and feel. Improvements where made to pagination aspects of the tables.

Add the ability to Duplicate a task

When the user wants to duplicate a task, they can simply access the details page of a Task and select the Duplicate button at the top of the page.

Provide the ability to move a task, issue, risk to another Project

When a user wants to move a Task, Issue, Risk, or Note to another Project, they can simply access the details page of the Task, Issue, or Risk and select a different Project.

Update the Summary Issues stats to include Associated Task Categories

The Overview and Summary pages have been updated to more easily access and read summary data.

Change assigned users to a RACI (Responsible, Accountable, Consulted, Informed) model

On Tasks, Issues, and Risks, Assigned Users have been migrated to be to use the RACI model.

As a result, there will be:

  • one user Responsible

  • one user Accountable

  • many users Consulted

  • many users Informed

Because there were existing assigned users a migration of these users will occur in conjunction to this release:

  • the first assigned user will be made the Responsible user

  • the second assigned user, if one exists, will be made the Accountable user

  • the remaining assigned users, if any exist, will be add to the Consulted users
