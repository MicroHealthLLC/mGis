What's New?

The February 5, 2021 release is primarily focused on improving the user interfaces that are associated with performing CRUD (Create, Read, Update, Delete) within mGIS. The Overview / Summary / Stats pages were updated for easier readability and the addition of requested information. The Map of the Map view was updated to resolve some inconsistencies, but more work is needed to maintain the context of the map as mGIS is being used. Some basic duplicate and move functionality was introduced, but more advanced functionality for copy / move will be include in future releases. Some performance enhancements were done as well.

Online Help Home page with access to the release plans

A new online help functionality is being added to mGIS. Though still under construction, this functionality can be access by selecting Help on the main menu in the upper right corner of mGIS. Selecting Help will invoke the mGIS Online Help site:

From this site the user will be able to access:

  • The Admin Panel User's Guide (still under construction)

  • The Client Panel User's Guide (still under construction)

  • What's New in the current release

  • What's Coming in future releases

  • What was included in Previous Releases

Rename Projects to Programs and Facilities to Projects

To provide a more broad usage of mGIS:

  • all references to Project(s) were changed to Program(s)

  • all references to Facility / Facilities were changed to Project(s)

Update the workflow to be consistent across CRUD (create, read, update, delete) activities

The Facility Manager has been deprecated because the Sheet view has been the preferred view of users.

The details pages for Tasks, Issues, and Risks under the Map and Sheet views have been updated for consistency. Additionally, these pages have been updated to improve the user experience by grouping the information under tabs so less if any scrolling is necessary.

Update implementation of records shown per page to be consistent throughout

All the tables throughout mGIS now have a consistent look and feel. Improvements where made to pagination aspects of the tables.

Add the ability to Duplicate a task

When the user wants to duplicate a task, they can simply access the details page of a Task and select the Duplicate button at the top of the page.

Provide the ability to move a task, issue, risk to another Project

When a user wants to move a Task, Issue, Risk, or Note to another Project, they can simply access the details page of the Task, Issue, or Risk and select a different Project.

Update the Summary Issues stats to include Associated Task Categories

The Overview and Summary pages have been updated to more easily access and read summary data.

Change assigned users to a RACI (Responsible, Accountable, Consulted, Informed) model

On Tasks, Issues, and Risks, Assigned Users have been migrated to be to use the RACI model.

As a result, there will be:

  • one user Responsible

  • one user Accountable

  • many users Consulted

  • many users Informed

Because there were existing assigned users a migration of these users will occur in conjunction to this release:

  • the first assigned user will be made the Responsible user

  • the second assigned user, if one exists, will be made the Accountable user

  • the remaining assigned users, if any exist, will be add to the Consulted users
